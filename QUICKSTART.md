# QUICKSTART — запуск на новом устройстве

> Данные (товары/клиенты/заказы) **не** хранятся в git — они тянутся из BAS
> автоматически при старте (~5 сек). В git только код. Поэтому нужен `.env`
> (его передаём отдельно — там ключи, в git его нельзя).

## Новое устройство (первый раз)

```bash
# 1. Забрать код
git clone <repo-url> svy_ai_agent
cd svy_ai_agent

# 2. Положить .env (скопировать с рабочего устройства — НЕ из git)
#    Должна стоять PostgreSQL. Проверка: psql --version

# 3. Запуск одной командой — сам создаст venv, поставит зависимости,
#    создаст БД, применит схему и стянет данные из BAS:
./run.sh start          # macOS / Linux
#   Windows: двойной клик run.bat  (или: run.bat)

# 4. Открыть дашборд и подключить Telegram по QR:
#    http://127.0.0.1:8000/dashboard  →  Налаштування → «Підключити через QR»
```

## Каждый день / после изменений

```bash
git pull
./run.sh restart        # macOS / Linux   (Windows: run.bat)
./run.sh status         # проверить, что всё ОК
```

## Полезное

```bash
./run.sh logs           # последние строки лога  (live: ./run.sh logs -f)
./run.sh stop           # остановить
./run.sh test           # прогнать тесты
```

## Если данных нет / по нулям

1. `.env` на месте и в нём `USE_MOCK=false` + верный `DATABASE_URL`?
2. PostgreSQL запущен? (`pg_isready` или `psql -l`)
3. BAS доступен с этого устройства? (`./run.sh logs` → искать `[SYNC]`)
4. Telegram подключён по QR? (без него бот не отвечает и не шлёт КП)

> Хотите, чтобы оба устройства видели **одни и те же** данные без пересинка —
> укажите в обоих `.env` один общий `DATABASE_URL` (облачный Postgres).
